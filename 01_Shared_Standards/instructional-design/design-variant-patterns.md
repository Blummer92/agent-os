# Design Variant Patterns

## Purpose

Use these rules for reusable visual systems, variant families, and repeatable
style patterns that should stay consistent across future classroom materials.

## Pattern Promotion Rule

Promote a new pattern here only after it proves useful across multiple runs.
Preserve recognizable visual patterns unless there is a clear instructional
reason to change them.

## Variant Families

A variant family should define:

- name
- use case
- typical materials
- notes or constraints

Use variant families for repeatable worksheet, slide, handout, checklist, or
reflection structures.

## Icons

- Use icons only for wayfinding or task clarity.
- Keep icon meaning consistent across related materials.
- Avoid decorative icons.
- Do not mix icon styles without an instructional reason.
- Reuse approved icon families or sources when available.

## Labels

- Use short, functional labels.
- Reuse the same label for the same student action.
- Prefer student-facing wording when students will read it.
- Standardize labels for repeated actions, checks, examples, and reflections.

## Layout Variants

- Reuse proven layouts.
- Keep directions, examples, practice, and reflection in predictable places.
- Preserve classroom-scannable structure over novelty.
- Use worksheet and slide variants only when they make task flow clearer.

## Color and Emphasis

- Use accent colors consistently; each accent should have a clear role.
- Reuse highlight styles, borders, callouts, and emphasis patterns.
- Avoid adding color or decoration that increases cognitive load.

## Component Patterns

Keep reusable blocks consistent when they appear across materials:

- checklist boxes
- worked example cards
- reflection boxes
- vocabulary callouts
- task cards
- comparison frames
