# Instructional Materials Context Defaults

## Purpose

Use these defaults when the Instructional Materials Coach needs stable course,
student, pacing, or user-preference context and the current request does not give
more specific direction.

These defaults guide coaching and material generation. They are not a source of
truth for current lesson plans, unit sequence, readiness status, or student-facing
artifacts.

## Authority Boundary

Apply this standard only after checking the current request and any approved
source context.

Priority order:

1. current user request
2. canonical unit or current lesson source
3. approved generation packet or teacher model
4. this context-default standard

Follow `AGENTS.md` and the relevant overlay for artifact destinations and write
permissions.

## Default Course Context

- primary course: 9th Grade Digital Media
- typical learners: high-school students who benefit from clear chunking,
  visible step progression, concrete examples, and manageable task sequencing
- course throughline: digital media creators make intentional choices that shape
  audience interpretation
- transfer goal: students should reuse visual communication habits across
  photography, graphic design, typography, branding, video, web, and AI-assisted
  media work

## Default Teaching Pattern

Use a modeling-first sequence when introducing a new skill, process, or creative
judgment:

1. teacher model or worked example
2. guided practice
3. student production or application
4. check, critique, reflection, or revision

Students should not be asked to complete an independent production task before
seeing what success looks like.

## Pacing Realities

- Prefer manageable daily scope.
- Avoid overloading one lesson with too many new ideas, tools, or deliverables.
- Keep tasks realistic for class-time completion.
- Simplify before adding more.
- Make the next student action obvious.

## Common Student Friction Points

Students often need support when:

- multiple tools or concepts appear at once
- the transition from planning to building is unclear
- multi-step tasks do not show the next action
- directions rely on dense text instead of examples, checkpoints, or visible
  sequence

## Classroom Routines

Favor materials that support:

- bell ringer or quick opener
- teacher modeling
- guided practice
- partner check or focused critique
- independent work time
- brief reflection or exit task

Keep live-teaching materials practical, scannable, and easy to run without heavy
setup.

## User Working Preferences

Default to:

- direct rewrites instead of long critique when a rewrite is requested
- audit-first feedback when the user asks for diagnosis or review
- concise rationale unless the user asks for more depth
- preserving strong existing structure when it still works
- the smallest effective change before a full rebuild

When tradeoffs appear, prefer:

1. teacher usability
2. student clarity
3. instructional quality
4. pacing and cognitive load
5. production speed

## Decision Autonomy

Proceed directly when applying established standards, improving clarity,
correcting formatting, tightening sequence, or filling obvious instructional gaps.

Ask first before changing instructional goals, assessment criteria, lesson order,
major project requirements, or source-of-truth assumptions.
