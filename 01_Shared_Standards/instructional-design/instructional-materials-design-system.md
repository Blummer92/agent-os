# Instructional Materials Design System

## Purpose

Use this standard to choose and shape classroom material types. Each material
should have a clear purpose, use case, structure, length, writing load, visual
need, teacher note, and quality check.

## Material Type Defaults

| Type | Use When | Default Structure |
|---|---|---|
| Build-Day Worksheet | Students are actively making or practicing. | Goal -> Plan -> Build -> Check -> Fix -> Explain. |
| Reflection Worksheet | Students look back on choices or process. | What I made -> What worked -> What changed -> What next. |
| Peer Feedback Sheet | Students give focused critique. | Notice -> Name -> Suggest -> Next step. |
| Classroom Poster | Students need a repeated visual reference. | Title -> instructional visual -> key points -> takeaway. |
| Tool Menu | Students need just-in-time tool help. | Tool -> Use it when -> Try -> Check. |
| Choice Board | Students need structured options. | Shared goal -> options -> completion target. |
| Mini Game | Interaction improves practice or review. | Target -> rules -> play loop -> debrief. |
| Vocabulary Cards | Students need term recognition and use. | Term -> meaning -> example -> optional visual cue. |
| Portfolio Prompt | Students explain process and growth. | What I made -> choices -> improvement -> evidence. |
| Rubric / Checklist | Students need success criteria. | Criteria -> indicators -> mark or note progress. |
| Troubleshooting Guide | Students solve recurring process problems. | Problem -> cause -> try -> check. |

## Game Format Rules

Use games only when interaction improves learning more than a simple reference or
tool guide. Prefer a tool guide when students mainly need step-by-step support,
a new process is being introduced, setup must be minimal, or game mechanics would
distract from the learning goal.

Useful formats include sort cards, match cards, spot-the-problem, before/after
design challenges, scavenger hunts, peer review bingo, vocabulary taboo, fix-this
design challenges, and gallery walk prompts.

## Dashboard Source Questions

When Notion is the control center, the source record should help answer:

- What unit and lesson is this for?
- What material type and template are needed?
- What images, vocabulary, and misconceptions matter?
- What should students produce by the end?

Recommended source areas include Unit Guides, Lesson Briefs, Templates, Visual
Asset Prompts, Vocabulary Bank, Student Skill Progression, Common Misconceptions,
and Finished Materials Archive.

## Image Asset Context

For image-based instructional assets, capture title, asset type, connected
assets, unit, lesson, learning goal, student task, teacher modeling use, source
material, version, date, reuse/new decision, related images, cognitive-load
concern, and accessibility concern.

Use this context to keep visuals tied to the lesson sequence, student task,
teacher use, and accessibility needs.

## Final Quality Check

Before using a worksheet, poster, game, tool guide, or related material, check:

- Is it too wordy?
- Are directions clear?
- Does the visual do instructional work?
- Does the student know what to do first?
- Can the teacher use it without explaining every box?
- Does it match the lesson sequence?
- Is it accessible and readable?
