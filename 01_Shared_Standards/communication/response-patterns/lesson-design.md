# Lesson Design Response Pattern

## Name

Lesson Design

## Purpose

Support classroom planning with short, usable decisions that preserve instructional alignment and reduce unnecessary detail.

## Status

experimental

## Default Modules

1. Decision
2. Source Note, only when the answer depends on Notion or repository evidence
3. Classroom Fit
4. Keep / Change
5. Next Step

## Max Length Guidance

- Target: 8 to 12 bullets or fewer.
- Focus on the part of the lesson the user asked about.
- Use a one-line source note when the answer is based on Notion, Drive, Sheets, or GitHub context.
- Do not write a full lesson plan unless requested.

## When To Use

- The user asks if a lesson idea works.
- The user asks how to simplify, align, or sequence instruction.
- The user is testing student-facing clarity.
- The user needs a teaching move, not a full artifact.

## When Not To Use

- The user asks for a complete slide deck, worksheet, rubric, or handout.
- The request belongs to a governed repository implementation workflow.
- The user asks for source-heavy research.
- The user only needs a quick yes/no recommendation.

## Example Response

**Decision**
Keep Pete Eckert as the hook, but teach only one formal word: `intentional`.

**Source Note**
Checked the Notion-grounded unit direction; not live-verified for readiness changes.

**Classroom Fit**
This fits a 50-minute launch because students can learn the other words through looking, talking, and trying the photo task.

**Keep / Change**
Keep: Pete Eckert, notice/think/wonder, blind-planning task.
Change: remove a separate vocabulary-definition section.

**Next Step**
Add teacher talk that uses `light`, `blur`, `angle`, and `distance` while pointing to specific parts of the image.

