# Quick Decision Response Pattern

## Name

Quick Decision

## Purpose

Give the user a fast recommendation when the main need is to choose, approve, reject, or move to the next step.

## Status

experimental

## Default Modules

1. Decision
2. Why
3. Next Action

## Max Length Guidance

- Target: 3 to 6 bullets total.
- Avoid long examples.
- Do not include background unless the user asks for it.

## When To Use

- The user asks which option is best.
- The user asks whether to proceed.
- The user is overloaded and needs a clear next move.
- The answer can be useful without a full analysis.

## When Not To Use

- The user explicitly asks for deep research.
- The task requires implementation evidence.
- The response must include a governed final report.
- The user needs a complete lesson, artifact, or specification.

## Example Response

**Decision**
Use vocabulary in context, not a separate definition list.

**Why**
Students need to use the words while looking at photos. A glossary would slow the lesson down.

**Next Action**
Revise the lesson so the teacher models the words `intentional`, `light`, `blur`, `angle`, and `distance` during photo discussion.

## Feedback Notes

```text
Did this answer help me decide quickly?
Was anything unnecessary?
Was the next action clear?
Keep / revise / reject:
```
