# Governance

Governance files define source-of-truth, authorization, memory, and standards-change rules.
These rules apply before any agent-specific overlay.

## Current baselines

- `agent-os-governance-v1-baseline.md` is the canonical Agent OS Governance v1.0 baseline for dashboard ownership, readiness vocabulary, production routing, production authorization, and governance change policy.

During pilot review, `agent-os-advisory-mode.md` is the default for day-to-day low-risk work.
It lets agents proceed with read-only, local-only, planning, drafting, QA notes, summaries, code review, local specs, and documentation drafts while preserving strict gates for risky actions.

## Documentation index

- `documentation-dependency-map.md` is the index for discovering existing documentation, ownership, dependency graph, and recommended reading paths. It points back to canonical documents and is not itself a source of truth.
