# Engineering Standards Framework

## Purpose
Provide inherited standards for coding and automation agents.

## Core Rules
- Review existing files, tests, known bugs, patterns, and release checklist before coding.
- Prefer modular components with one responsibility.
- Keep orchestration thin; put business logic in components.
- Add tests for recurring bugs and behavior changes.
- Report files changed, tests run, bugs fixed, docs updated, Notion updates, and memory recommendations.
- Default live workspaces to read-only unless explicitly authorized.
