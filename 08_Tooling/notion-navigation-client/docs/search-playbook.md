# Search Playbook

Repeatable Notion search patterns that help agents find the right Digital Media content quickly. This is retrieval guidance only; verify live Notion before readiness, status, ownership, curriculum, or governed-field decisions.

## Retrieval approach

1. Start with the most likely source-of-truth page or database.
2. Prefer named dashboards and linked databases before broad keyword-only search.
3. When dashboards show summaries but not active rows, follow links back to the source database or canonical item page.
4. Distinguish authoritative pages from derived dashboards, archived copies, and duplicate views.
5. Record only durable search patterns likely to help again.

## Use-case routing

| User question | Start with |
|---|---|
| Which source should I trust? Can generation proceed? | Digital Media Source Control Dashboard |
| Is this unit approved to generate? What is the primary unit-level blocker? | Unit Readiness Workspace |
| What teacher modeling is missing before slides can proceed? | Modeling Coach Workspace |
| Is student evidence strong enough? What evidence gap blocks progress? | Evidence Coach Workspace |
| What is the smallest next packet move or packet-level blocker? | Generation Packet Workspace |
| What is blocked, who owns it, or what needs human action? | Digital Media Curriculum Command Center or Human Action View, then route back to owner dashboards. |
| Which review record needs routing or handoff attention? | Digital Media Source Review Dashboard and Digital Media Source Review Records. |
| What is the canonical unit name or alias? | Canonical Digital Media Unit Registry. |

## Durable heuristics

- Treat command centers, human action views, and dashboards as summary/routing surfaces until proven otherwise.
- Treat pages or databases explicitly naming source control, registry, workspace, coach, or source of truth as higher-authority candidates, then verify ownership notes.
- When search returns both a dashboard page and its underlying database, prefer the database for record-level fields and the page for use guidance and boundaries.
- When a command-center queue says "route to specialist" or "move packet forward," use the linked owner workspace rather than deciding from the summary surface alone.
- For active-work summaries, look for linked views and owner-routing pages before reconstructing records from search results.
- When row-level filtered contents are hidden, preserve the source page or database link instead of inferring hidden active rows.

## Record useful patterns like

- Owner-field naming differences.
- Where active-work links usually appear.
- Which pages tend to be summary-only versus record-level.
