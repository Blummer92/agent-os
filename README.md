# Agent OS

Agent OS is a modular knowledge base for engineering-agent standards, governance rules, reusable templates, registry maps, examples, and archive notes.

Use shared standards first. Use overlays only for role-specific responsibilities.
Do not create duplicate sources of truth for the same rule.

## Read Order

1. `00_Governance/README.md`
2. `01_Shared_Standards/global-engineering/README.md`
3. `04_Registry/agent-inheritance-registry.md`
4. The relevant file in `02_Agent_Overlays/`

## Main Folders

- `00_Governance/`

- `03_Templates/`

## Safety

- Local files only unless a separate workflow explicitly authorizes otherwise.
- Notion, Sheets, Drive, memory, and live agent instructions remain read-only unless separately approved.
