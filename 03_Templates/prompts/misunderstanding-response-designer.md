# Misunderstanding Response Designer Template

## Purpose

Use this template when likely student misunderstandings need to become concrete
teacher responses, support moves, checks for understanding, or model revisions.

## Use When

- The teacher asks what students will probably get wrong.
- A model, worksheet, prompt, or transition may create confusion.
- Likely confusion needs precise teacher response lines or moves.
- The user wants a misunderstanding chart for one lesson section.

## Do Not Use When

- The main need is a full section-by-section lesson audit.
- The main need is extended teacher talk for the whole lesson.
- The modeling sequence itself is missing or misaligned.
- The objective, task, or key modeling moment is too unclear to ground the answer.

## Core Workflow

1. Identify the exact teaching moment.
2. State what students are supposed to think, notice, produce, or do.
3. Compare that with the likely incorrect interpretation or move.
4. Separate issue, cause, visible evidence, and teacher response.
5. Prefer prevention when a small revision would avoid the problem.
6. Add exact teacher wording when the better language is clear.
7. Include a quick check that shows whether the fix worked.

## Default Output

Use 2-5 high-probability misunderstandings, not an exhaustive list.

For each misunderstanding, include:

- likely misunderstanding
- why it may happen
- what the teacher would notice
- best teacher move
- optional prevention revision
- quick check for understanding

## Output Variants

### Fast Coaching Response

Use 2-4 high-leverage misunderstandings with the issue, response move, and one
short prevention revision or teacher line.

### Lesson-Section Map

Use one row or bullet per misunderstanding with the full default output.

### Revision-First Response

Start with the exact revision to the model, prompt, example, or explanation;
then name the misunderstanding it prevents and add a live support move if needed.

## Decision Rules

- Focus on misunderstandings that would disrupt the next student task.
- Avoid vague labels like "students may be confused."
- Always include the instructional response, not only the diagnosis.
- Combine misunderstandings with the same cause when that improves usability.
- Mark assumptions when source evidence is weak.
- Say directly when teacher wording, model design, or task alignment causes the issue.

## Quality Bar

A strong response makes the teacher think: that is the likely mistake, I know
what to say or change next, and the fix matches the real task.
